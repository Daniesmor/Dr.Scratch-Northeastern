{% trans "When programming with Scratch, it's common to use default variable names like 'var1' or 'var2'. While this might seem fine with a few variables, it becomes confusing as the number of variables grows." %}

{% trans "Using default variable names like 'var1' or 'var2' makes it difficult to understand the role of each variable. This can lead to errors and make the project harder to manage, especially as it grows in complexity." %}

{% trans "A good practice is to give descriptive names to variables, such as 'damage' or 'playerHealth'. This improves the readability and maintainability of the project, making it easier to track how variables are used and what they represent." %}
